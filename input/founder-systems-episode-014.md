Founder Systems Episode 014

This reference episode demonstrates the orchestration layer on top of the Bonfyre floor.

The floor keeps the artifact graph small and deterministic:
- one source object
- one brief
- one renderable surface

The orchestration layer only earns its place when it fans the same source into more useful outputs without changing the object model:
- tighter release framing
- narrated summary potential
- packaging for distribution
- clearer publish-ready show notes

The point is not to hide Bonfyre behind prompting.
The point is to let Bonfyre stay composable while the orchestration layer silently chooses when extra output fanout is actually worth the added cost.
